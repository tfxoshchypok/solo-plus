# solo.plus

Десктопний застосунок для проєктування **електрощитків (ЕРЩ)**. Електрик описує своїх споживачів — програма рахує струми, підбирає кабель, розставляє апарати на DIN-рейках і генерує документацію (схему, список покупок, наклейки). Усе працює **локально**, без хмар та акаунтів.

## Що це вирішує

Раніше струми, переріз кабелю й розкладку апаратів електрик рахував у голові та на папері. solo.plus бере цю рутину на себе: ти описуєш щиток у звичних термінах, а інженерне ядро виконує розрахунки й перевірки в реальному часі.

## Головні поняття

- **Проєкт = один щиток = один файл (`.splus`).** Як `.docx` у Word — проєкти незалежні, файл легко віддати чи заархівувати.
- **Група = одна лінія/відгалуження** = один автомат + кабель + споживачі на ньому. Електрик мислить групами («розетки кухні», «освітлення», «духовка»), а не окремими розетками. Кожна група — це абстрактний `spec` (апарат, номінал, крива, полюси — **без бренду**) плюс необов'язкове навантаження (Вт, фази, cosφ), з якого рахується струм і кабель.
- **Прив'язка до виробника (`productBinding`)** — окремий шар: інженерне ядро працює з абстракцією (ABB ≠ Schneider не впливає на фізику), а артикули й ціни накладаються зверху.
- **Розкладка (placement)** — де апарат стоїть на рейці, у **модулях** (1 модуль = 17.5 мм), а не в пікселях, тож зум не псує дані.

## Потік роботи

1. Створити/відкрити проєкт (щиток)
2. Завести **групи** (апарат + номінал + навантаження)
3. Програма рахує струми й переріз кабелю
4. Розкласти групи по рейках (drag-and-drop, перевірки в реальному часі)
5. Згенерувати документи

## Технології

- **Vue 3** + **Pinia** — UI та стан
- **Konva** — інтерактивне полотно розкладки апаратів
- **Naive UI** — компоненти інтерфейсу
- **Dexie (IndexedDB)** — локальне сховище
- **Ajv** — валідація схем даних
- **Neutralino** — пакування в десктопний застосунок
- **Vite** — збірка; **Vitest** + **Playwright** — тести; **TypeScript** (через JSDoc, `--checkJs`)

## Структура

```
src/
  calc/        — інженерні розрахунки (струми, кабель)
  components/  — Vue-компоненти UI
  db/          — Dexie / IndexedDB
  editor/      — полотно розкладки (Konva)
  export/      — генерація документів
  i18n/        — локалізація
  model/       — модель даних щитка
  project/     — робота з проєктами (.splus)
  stores/      — стан Pinia
  styles/      — стилі та тема
  workers/     — фонові обчислення
schema/        — JSON-схеми даних
tests/         — unit / e2e тести
```

## Розробка

```bash
npm install          # встановити залежності

npm run dev          # запуск у браузері (Vite)
npm run neu:dev      # запуск як десктоп-застосунок (Neutralino)

npm run test         # unit-тести (Vitest)
npm run test:e2e     # e2e-тести (Playwright)
npm run typecheck    # перевірка типів (JSDoc + tsc)

npm run build        # збірка веб-версії
npm run neu:build    # збірка десктоп-релізу
```
